Codex, Meet Bookmaker

The open-source bookmaking tool from Macmillan Publishers.

What Is Bookmaker?

Bookmaker is a collection of tools and scripts to make books out of Word .docx files and HTMLBook HTML files (it can accept either type of file as input). It was developed as part of a pilot project to try to make books with almost no budget, and it is currently in use today at Macmillan.

Requirements

Macmillan Word template

In order to get well-structured output files, you need at least somewhat well-structured input files. We created a set of styles to apply to the parts of a Word document, to provide the structure we need. Get the template here.

HTMLBook HTML

Just like with the Word files, we need to have some rules around how the HTML is structured, so bookmaker knows what to do with it. We used O'Reilly's HTMLBook spec, which you can read about here.

 

Ruby

This open-source programming language lies at the heart of bookmaker.

Python

Also open-source, python assists the ruby processes.

HTMLBook

We use O'Reilly's HTMLBook toolkit to create the EPUB files (why reinvent the wheel, right?).

Java and Saxon

Turns .docx into HTML. (Help us get rid of this--see the Project Ideas below.)

An XSL1 Processor

Turns HTML into an EPUB.

Download Sample Files

We put together a little sample book that you can use with bookmaker. It includes a Word .docx file with the Macmillan template already applied, an HTMLBook version of the same book, a cover file, the metadata JSON file, and some CSS to make the PDF a little bit prettier (just a little bit). Drop either the .docx or the HTML file into your conversion folder along with the cover and JSON, and follow the instructions below to convert your book to PDF and EPUB.

Which files should I use?

.docx

If you installed all the dependencies, inclduing Saxon and Java, then the Word file is for you.

HTML

If you decided not to install Saxon and Java, you can still run bookmaker--just use the HTML file.

Run Bookmaker

To run bookmaker, place your book files in the "convert" folder, and then open a Terminal or Command Prompt window and run the file called "makebook" (you can find it inside the bookamker-master folder) on your .docx or HTML file, like this:

Windows:

makebook.bat /Users/You/Desktop/convert/alice_in_wonderland.docx

Mac:

makebook.sh /Users/You/Desktop/convert/alice_in_wonderland.docx

Tip: Make sure to include the full path to the .docx or HTML file in your command. You can also just drag the file onto your Terminal or Command Prompt window and the full path will be auto-completed for you.

Project Ideas

Want to build something around bookmaker? Here are some ideas!

.docx to HTML, the open version

Bookmaker currently uses XSLT2 to transform .docx to HTML (we use Java and Saxon to run these scripts). Write a script that can transform the tagged .docx file to HTML in the same way, but using a more open language like Python or JavaScript (with Node or some other non-browser-based JavaScript utility).

Requires: Document transformation language like Javascript, or Python.

Any Word file

Right now .docx files must be styled with the Macmillan Word template in order to convert well. Write an extension to re-map a different template's styles to the Macmillan styles. Extra challenge: make it scalable--that is, make it so that the extension can be reused for any set of styles, and doesn't require a developer to add support for new styles.

Requires: Programming language like Ruby or Python (or even JavaScript?)

Any HTML to HTMLBook

Bookmaker requires input HTML files to be valid HTMLBook. Write an extension to convert plain HTML files to HTMLBook.

Requires: Document transformation language like XSLT, Javascript, or Python

Beautiful books

Bookmaker doesn't ship with any CSS files to make the output prettier. Create a library of well-designed CSS templates for EPUB and PDF files. Tips: Writing CSS for EPUBs is a lot like writing CSS for very basic websites. Here's some help getting started writing CSS for PDFs.

Requires: CSS, good design sense

Dropbox + bookmaker

Using the Dropbox API, create an app that can send files between DropBox and bookmaker.

Requires: DropBox API, programming language that can work the API (like Ruby or Python), some sort of front end.
